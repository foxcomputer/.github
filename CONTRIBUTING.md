# Contributing to foxcomputer

Hey there, fellow bot builder / pixel wrangler / chaos gremlin. Welcome! We're happy you're here and thinking about contributing.
This document tells you everything you need to know to contribute without summoning any cosmic horrors.

## ️ Ways to Contribute

There are lots of ways to help out:

* Fix bugs (even the embarrassing ones)
* Improve documentation
* Add new features (responsibly, pls)
* Refactor cursed code
* Report issues with clear repro steps
* Suggest improvements and enhancements

## Setting Up Your Environment

1. **Fork** the repository.
2. **Clone** your fork.
3. Make sure you have the correct dependencies installed.
4. Create a new branch for your changes.

Example workflow:

```
git checkout -b feature/my-awesome-feature
```

## Running Tests

All tests must pass before submitting a pull request. If adding new functionality, please add relevant tests.

## Submitting a Pull Request

When you're ready:

1. Commit your changes with a clear and descriptive message.
2. Push your branch:

```
git push origin feature/my-awesome-feature
```

3. Open a Pull Request.
4. Fill out the PR template.
5. Wait for feedback.

## PR Guidelines

* Keep PRs focused. No kitchen-sink PRs.
* Make sure your code follows the project's style guidelines.
* Write helpful commit messages.
* Squash commits if asked.

## ️ Code of Conduct

By participating, you agree to follow the project's Code of Conduct.

## Questions?

Open an issue or start a discussion. We're friendly, promise (unless you break the build on Friday at 6 PM).
